# 🇰🇷 한국어 개발/기술 리소스 컬렉션

이 저장소는 **한국어로 제공되는 쓸만한 개발/기술 리소스**를 모아서 정리하는 저장소입니다.  
사이트, GitHub 레포지토리, 블로그, 동아리/커뮤니티, 교육 프로그램 등 유용한 자료를 한 곳에서 찾아볼 수 있도록 만드는 것이 목표입니다.

> ⚠️ PR/Issue 환영  
> 새로운 리소스 제안, 잘못된 정보 수정, 카테고리 구조 개선 등 어떤 기여든지 환영합니다.

---

## 📂 디렉토리 구조

이 저장소는 **Markdown(.md) 파일만으로 구성**되며, 기본 구조는 다음과 같습니다.

```text
.
├─ README.md          # 저장소 소개 및 기여 가이드
├─ websites/          # 웹 서비스/리포트/레퍼런스 사이트
│  └─ dev.md          # 개발/FE/BE/일반 기술 웹사이트 모음
├─ repositories/      # GitHub, GitLab 등 코드/모음 레포
├─ blogs/             # 개인/팀/기업 기술 블로그
├─ communities/       # 동아리, 스터디, 온라인 커뮤니티
├─ education/         # 강의, 부트캠프, 교육 기관
└─ etc/               # 위 어디에도 딱 맞지 않는 기타
```

필요에 따라 디렉토리/파일을 자유롭게 추가·수정할 수 있습니다.

## 🗂 분류 기준

리소스는 대략 다음 기준을 참고해 분류합니다.

- **형태별**
  - 사이트 / Git 리포지토리 / 블로그 / 동아리 / 커뮤니티 / 강의 / 기타
- **분야별**
  - 웹 / 프론트엔드 / 백엔드 / 모바일 / 데이터 / AI / 보안 / 게임 / CS 이론 등
- **대상 레벨**
  - 입문 / 초급 / 중급 / 고급
- **언어**
  - 기본적으로 한국어 중심
  - 필요 시 한국어 지원이 좋은 영어 리소스도 포함 가능

각 `.md` 파일 상단에 어떤 리소스를 모으는 문서인지 짧게 설명을 남깁니다.

## 🔖 기본 포맷

각 리소스는 최대한 **한 눈에 이해 가능하도록** 아래 포맷을 권장합니다.

```markdown
- [이름](https://link.example)
  - **설명**: 한 줄~두 줄 정도로 핵심 요약
  - **분류**: 웹 / 프론트엔드 / 백엔드 / AI / 동아리 / 커뮤니티 / 블로그 / 강의 / 리포트 ...
  - **레벨**: 입문 / 초급 / 중급 / 고급
  - **비고**: 지역, 업데이트 주기, 가입 조건, 특징 등 (선택)
```

필요 없다면 `비고`는 생략해도 됩니다.

## 🤝 기여 방법

이 저장소는 **모든 기여를 환영**합니다.

1. **Fork & Branch**
   - 저장소를 fork 한 뒤 작업용 브랜치를 생성합니다.
2. **적절한 MD 파일 선택**
   - 예: 웹 리포트/사이트 → `websites/dev.md`
   - 예: 개인 블로그 → `blogs/individual.md`(생성 후 사용) 등
3. **템플릿에 맞춰 항목 추가**
   - 위의 “기본 포맷”에 맞게 리소스를 추가합니다.
4. **PR 생성**
   - 변경 내용을 간단히 설명하는 PR을 생성합니다.
   - 가능하다면 “왜 유용한지”, “어떤 사람에게 특히 도움이 되는지”를 적어 주세요.

### PR 제목 예시

- `Add: 2025 State of Frontend JavaScript 리포트`
- `Add: OO대학교 알고리즘 동아리`
- `Add: 한국어 딥러닝 강의 모음 레포지토리`
